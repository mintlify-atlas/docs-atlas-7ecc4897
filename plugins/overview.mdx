## Architecture

nvim-luffy uses **lazy.nvim** as its plugin manager, providing a modern, fast, and feature-rich plugin management system. The configuration follows a modular structure where each plugin or group of related plugins has its own configuration file.

### Directory Structure

```
lua/leyland/
├── lazy.lua                 # Plugin manager bootstrap
└── plugins/
    ├── init.lua            # Core plugins
    ├── colorscheme.lua     # Theme configurations
    ├── telescope.lua       # Fuzzy finder
    ├── nvim-tree.lua       # File explorer
    ├── nvim-cmp.lua        # Completion engine
    ├── treesitter.lua      # Syntax highlighting
    ├── harpoon.lua         # File navigation
    ├── flash.lua           # Motion plugin
    ├── neogit.lua          # Git client
    ├── diffview.lua        # Diff viewer
    ├── alpha.lua           # Dashboard
    ├── snacks.nvim         # UI components
    ├── trouble.lua         # Diagnostics list
    ├── formatting.lua      # Code formatting
    ├── autopairs.lua       # Auto pairs
    └── lsp/
        ├── lsp.lua         # LSP configurations
        └── mason.lua       # LSP installer
```

<Note>
All plugins are lazy-loaded by default, improving Neovim's startup time significantly.
</Note>

## Plugin Loading Strategy

<Tip>
lazy.nvim automatically lazy-loads plugins based on events, commands, file types, and key mappings defined in each plugin's specification.
</Tip>

The plugin system uses two main import paths:

1. **`leyland.plugins`** - General plugins (UI, navigation, git, etc.)
2. **`leyland.plugins.lsp`** - LSP-related plugins

This separation keeps LSP configurations isolated and easier to manage.

## Features

- **Automatic Installation**: lazy.nvim bootstraps itself on first run
- **Version Locking**: `lazy-lock.json` ensures consistent plugin versions
- **Lazy Loading**: Plugins load only when needed
- **Update Checking**: Automatic check for updates (notifications disabled)
- **Change Detection**: Monitors plugin spec changes (notifications disabled)
