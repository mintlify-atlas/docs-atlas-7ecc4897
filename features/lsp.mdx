## Overview

nvim-luffy integrates Language Server Protocol (LSP) support through Mason and mason-lspconfig, providing automatic installation and configuration of language servers.

## Architecture

The LSP setup consists of three main components:

1. **mason.nvim** - Package manager for LSP servers, formatters, and linters
2. **mason-lspconfig.nvim** - Bridge between Mason and nvim-lspconfig
3. **nvim-lspconfig** - LSP configuration plugin

## Automatic Installation

LSP servers are automatically installed on first launch through mason-lspconfig:

<Code>
```lua
ensure_installed = {
  "ts_ls",
  "html",
  "cssls",
  "tailwindcss",
  "svelte",
  "lua_ls",
  "graphql",
  "emmet_ls",
  "prismals",
  "pyright",
  "eslint",
}
```
</Code>

<Note>
All configured servers will be automatically downloaded and installed when you first open a file of the corresponding type.
</Note>

## Mason UI

Mason provides a user interface to manage installed packages:

- **✓** - Package installed
- **➜** - Package pending installation
- **✗** - Package not installed

<Tip>
Open the Mason UI with `:Mason` to manually install, update, or remove language servers.
</Tip>

## Additional Tools

Beyond LSP servers, Mason also automatically installs development tools:

<Code>
```lua
ensure_installed = {

  "black",      -- Python formatter
  "pylint",     -- Python linter
  "eslint_d",   -- Fast ESLint daemon

```
</Code>

## LSP Capabilities

All LSP servers are configured with enhanced capabilities from nvim-cmp:

<Code>
```lua
local cmp_nvim_lsp = require("cmp_nvim_lsp")
local capabilities = cmp_nvim_lsp.default_capabilities()

vim.lsp.config("*", {
  capabilities = capabilities,
})
```
</Code>

This enables:
- Autocompletion with LSP suggestions
- Snippet support
- Advanced text editing capabilities

## File Operations Integration

LSP file operations are enhanced with `nvim-lsp-file-operations`, enabling:
- Automatic LSP updates when renaming files
- Import path updates when moving files
- Workspace-aware file operations
