# Quickstart Guide

Get up to speed with nvim-luffy in minutes. This guide covers the essential keybindings and workflows you'll use every day.

## First Launch

When you first open Neovim after installation:

1. **lazy.nvim** automatically bootstraps and installs all plugins
2. **Mason** downloads LSP servers and formatters in the background
3. **Treesitter** compiles syntax parsers for your languages

<Note>
  The first launch may show some errors - this is normal! Close and reopen Neovim once installation completes.
</Note>

After the initial setup, launch times are near-instant thanks to lazy loading.

## The Leader Key

The **leader key** is `Space`. Almost all custom keybindings start with pressing the spacebar.

<Tip>
  Think of the leader key as your "command prefix" - it unlocks all the powerful features.
</Tip>

## Essential Keybindings

### File Navigation

<CodeGroup>
```text Telescope (Fuzzy Finder)
<Space><Space>  Find files in current directory
<Space>/        Search text across project (live grep)
<Space>fr       Find recent files
<Space>fc       Find word under cursor
<Space>ft       Find TODO comments
<Space>fk       Find keymaps (discover more shortcuts!)
```

```text File Explorer
<Space>e        Toggle nvim-tree file explorer

# Inside nvim-tree:
l               Open file/folder
h               Close folder
a               Create new file
d               Delete file
r               Rename file
y               Copy file
p               Paste file
```

```text Harpoon (Quick File Marks)
<Space>H        Add current file to Harpoon
<Space>h        Open Harpoon quick menu
<Space>1-9      Jump to harpooned file 1-9
```
</CodeGroup>

### Window Management

```text
<Space>|        Split window vertically
<Space>sh       Split window horizontally
Ctrl+h/j/k/l    Navigate between windows (works with tmux!)
```

<Tip>
  The tmux navigation integration means you can move between Neovim splits and tmux panes with the same keybindings.
</Tip>

### Buffer Management

```text
<Space>bd       Close current buffer
<Space>ba       Close all buffers
```

### Git Operations

```text
<Space>gg       Open Neogit (Git interface)
```

Inside Neogit:
- `s` - Stage file/hunk
- `u` - Unstage file/hunk
- `c` - Commit (opens commit message buffer)
- `P` - Push
- `F` - Pull
- `?` - Show help

## LSP Features

When editing code files with LSP support (TypeScript, Python, Lua, etc.):

### Code Intelligence

```text
gd              Go to definition
gr              Show references
K               Show hover documentation
<Space>ca       Code actions
<Space>rn       Rename symbol
<Space>D        Type definition
[d              Previous diagnostic
]d              Next diagnostic
<Space>rs       Restart LSP server
```

### Completion

In insert mode, completions appear automatically. Navigate them with:

```text
Ctrl+j          Next completion
Ctrl+k          Previous completion
Ctrl+Space      Trigger completion
Tab             Select completion / expand snippet
Enter           Confirm completion
Ctrl+e          Close completion menu
```

## Telescope Deep Dive

Telescope is your primary navigation tool. Inside any Telescope picker:

```text
Ctrl+j/k        Move up/down in results
Ctrl+q          Send selected to quickfix list
Enter           Open selection
Esc             Close Telescope
```

### Typical Workflow

1. **Find a file**: `<Space><Space>`, type partial filename, `Enter`
2. **Search for text**: `<Space>/`, type search term, navigate with `Ctrl+j/k`
3. **Jump to definition**: `gd` when cursor is on a symbol
4. **Go back**: `Ctrl+o` (standard Neovim jump-back)

## Harpoon Workflow

Harpoon is perfect for working with a small set of frequently-accessed files:

1. Open a file you use often
2. Press `<Space>H` to "harpoon" it
3. Repeat for up to 9 files
4. Use `<Space>1` through `<Space>9` to instantly jump between them
5. Press `<Space>h` to see all harpooned files

<Tip>
  Harpoon is great for rapidly switching between implementation and test files!
</Tip>

## Editing Workflows

### Auto-formatting

Code is automatically formatted on save using:
- **prettier** for JavaScript/TypeScript/JSON/CSS/HTML
- **stylua** for Lua
- **black** and **isort** for Python

No keybinding needed - just save the file!

### Treesitter Text Objects

Treesitter enables intelligent text objects:

```text
vaf             Select around function
vif             Select inside function
vac             Select around class
vic             Select inside class
```

## Discovering More

### Built-in Help

```text
<Space>fk       Find keymaps - search all available shortcuts
:Lazy           Open plugin manager UI
:Mason          Open LSP server manager
:help <topic>   Neovim's built-in help
```

### Plugin-specific Help

Many plugins show a help menu when you press `?` inside them:
- Neogit: `<Space>gg` then `?`
- Telescope: `<Space><Space>` then `Ctrl+/` (shows mappings)
- nvim-tree: `<Space>e` then `g?`

## Common Workflows

### Starting a New Feature

<Steps>
  <Step title="Open your project">
    ```bash
    cd ~/projects/myapp
    nvim .
    ```
  </Step>

  <Step title="Find the file to edit">
    Press `<Space><Space>`, type filename, `Enter`
  </Step>

  <Step title="Navigate to the function">
    Use `/` to search, or `<Space>/` to search project-wide
  </Step>

  <Step title="Make your changes">
    Use LSP features:
    - `gd` to jump to definitions
    - `K` to see documentation
    - `<Space>ca` for code actions
    - Let auto-completion help you
  </Step>

  <Step title="Review changes">
    Press `<Space>gg` to open Neogit and see your diff
  </Step>

  <Step title="Stage and commit">
    In Neogit:
    - `s` to stage hunks
    - `c` to commit
    - Write message and save
    - `P` to push
  </Step>
</Steps>

### Debugging LSP Issues

<Steps>
  <Step title="Check if LSP is attached">
    Run `:LspInfo` to see active language servers
  </Step>

  <Step title="Verify server is installed">
    Run `:Mason` and check if the server is installed
  </Step>

  <Step title="Restart LSP">
    Press `<Space>rs` to restart the language server
  </Step>

  <Step title="Check LSP logs">
    Run `:LspLog` to see error messages
  </Step>
</Steps>

### Working with Multiple Files

1. **Harpoon for frequent files**: Mark your main files with `<Space>H`, jump with `<Space>1-9`
2. **Telescope for searching**: Use `<Space><Space>` when you need to find something
3. **LSP for navigation**: Use `gd` and `gr` to follow code references
4. **Splits for side-by-side**: Use `<Space>|` to view two files at once

## Editor Configuration

key settings (from `lua/leyland/core/options.lua`):

- **Leader key**: Space
- **Tab size**: 2 spaces
- **Line numbers**: Relative (easier for motions like `10j`)
- **Search**: Case-insensitive unless you use capitals
- **Clipboard**: Synced with system clipboard
- **Splits**: Open right/below (more natural)
- **No swap files**: Clean working directory

## Customization Hints

All keybindings are defined in:
- **Global**: `lua/leyland/core/keymaps.lua`
- **Plugin-specific**: Inside each plugin file in `lua/leyland/plugins/`

To customize:
1. Edit the relevant file
2. Save
3. Restart Neovim (or `:source %` to reload)

<Tip>
  Each keymap has a `desc` field - this is what shows up in `<Space>fk`!
</Tip>
