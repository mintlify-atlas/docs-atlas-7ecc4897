## What is lazy.nvim?

**lazy.nvim** is a modern plugin manager for Neovim that focuses on:

- Fast startup times through lazy loading
- Clean, declarative plugin specifications
- Built-in UI for managing plugins
- Automatic dependency resolution
- Git integration for updates

<Note>
lazy.nvim can improve Neovim startup time by loading plugins only when they're actually needed.
</Note>

## Bootstrap Configuration

The bootstrap code in `lua/leyland/lazy.lua` automatically installs lazy.nvim if it's not present:

```lua title="lua/leyland/lazy.lua"
local lazypath = vim.fn.stdpath("data") .. "/lazy/lazy.nvim"
if not vim.loop.fs_stat(lazypath) then
	vim.fn.system({
		"git",
		"clone",
		"--filter=blob:none",
		"https://github.com/folke/lazy.nvim.git",
		"--branch=stable", -- latest stable release
		lazypath,
	})
end
vim.opt.rtp:prepend(lazypath)
```

### What This Does

<Steps>
  <Step title="Define the lazy.nvim path">
    Sets `lazypath` to Neovim's data directory + `/lazy/lazy.nvim`
  </Step>
  <Step title="Check if lazy.nvim exists">
    Uses `vim.loop.fs_stat()` to check if the directory exists
  </Step>
  <Step title="Clone if missing">
    If not found, clones the stable branch from GitHub using git with blob filtering for faster download
  </Step>
  <Step title="Prepend to runtime path">
    Adds lazy.nvim to the runtime path so it can be required
  </Step>
</Steps>

## Setup Configuration

After bootstrapping, lazy.nvim is configured with plugin specifications:

```lua title="lua/leyland/lazy.lua"
require("lazy").setup(
  { 
    { import = "leyland.plugins" }, 
    { import = "leyland.plugins.lsp" } 
  },
  {
    checker = {
      enabled = true,
      notify = false,
    },
    change_detection = {
      notify = false,
    },
  }
)
```

### Configuration Breakdown

#### Plugin Imports

```lua
{ import = "leyland.plugins" }
{ import = "leyland.plugins.lsp" }
```

lazy.nvim will automatically load all Lua files from:
- `lua/leyland/plugins/` directory
- `lua/leyland/plugins/lsp/` directory

Each file should return a plugin specification table.

#### Options

| Option | Value | Description |
|--------|-------|-------------|
| `checker.enabled` | `true` | Automatically check for plugin updates |
| `checker.notify` | `false` | Don't show notifications for available updates |
| `change_detection.notify` | `false` | Don't notify when plugin specs change |

<Tip>
Disabling notifications keeps your editing experience clean while still checking for updates in the background.
</Tip>

## Plugin Specifications

Each plugin file returns a specification table. Here's an example from `telescope.lua`:

```lua title="lua/leyland/plugins/telescope.lua"
return {
  "nvim-telescope/telescope.nvim",
  branch = "0.1.x",
  dependencies = {
    "nvim-lua/plenary.nvim",
    { "nvim-telescope/telescope-fzf-native.nvim", build = "make" },
    "nvim-tree/nvim-web-devicons",
    "folke/todo-comments.nvim",
  },
  config = function()
    -- Plugin configuration goes here
    local telescope = require("telescope")
    telescope.setup({
      -- settings
    })
  end,
}
```

### Specification Fields

| Field | Description | Example |
|-------|-------------|----------|
| `[1]` | Plugin repository (first element) | `"nvim-telescope/telescope.nvim"` |
| `branch` | Git branch to use | `"0.1.x"` or `"main"` |
| `dependencies` | Other plugins this depends on | `{ "nvim-lua/plenary.nvim" }` |
| `config` | Function to run after plugin loads | `function() ... end` |
| `opts` | Options table passed to `setup()` | `{ ui = { ... } }` |
| `cmd` | Load on command | `"Telescope"` |
| `keys` | Load on keymap | `"<leader>ff"` |
| `event` | Load on event | `"BufEnter"` or `"VeryLazy"` |
| `ft` | Load on filetype | `"lua"` or `{ "lua", "vim" }` |
| `build` | Command to run after install/update | `"make"` |

## Lazy Loading Strategies

lazy.nvim provides several ways to lazy-load plugins:

### 1. Command-based Loading

```lua
return {
  "neogit",
  cmd = "Neogit", -- Loads when :Neogit is executed
}
```

### 2. Keymap-based Loading

```lua
return {
  "flash.nvim",
  keys = { -- Loads when these keys are pressed
    { "s", mode = { "n", "x", "o" } },
    { "S", mode = { "n", "x", "o" } },
  },
}
```

### 3. Event-based Loading

```lua
return {
  "nvim-autopairs",
  event = "InsertEnter", -- Loads when entering insert mode
}
```

### 4. Filetype-based Loading

```lua
return {
  "vim-python-pep8-indent",
  ft = "python", -- Loads only for Python files
}
```

### 5. Immediate Loading

```lua
return {
  "nvim-web-devicons",
  lazy = false, -- Load immediately on startup
}
```

<Note>
If no lazy-loading trigger is specified (cmd, keys, event, ft), the plugin is loaded immediately.
</Note>

## Plugin Directory Structure

The modular structure allows for clean organization:

```
lua/leyland/plugins/
├── init.lua              # Core utilities (plenary, tmux-navigator)
├── colorscheme.lua       # Theme configuration
├── telescope.lua         # Fuzzy finder
├── nvim-tree.lua         # File explorer
├── nvim-cmp.lua          # Completion
├── treesitter.lua        # Syntax highlighting
├── harpoon.lua           # File navigation
├── flash.lua             # Motion
├── neogit.lua            # Git client
├── diffview.lua          # Diff viewer
├── alpha.lua             # Dashboard
├── snacks.lua            # UI utilities
├── trouble.lua           # Diagnostics
├── formatting.lua        # Formatters (conform.nvim)
├── autopairs.lua         # Auto pairs
└── lsp/
    ├── lsp.lua           # LSP server configurations
    └── mason.lua         # LSP/tool installer
```

### Why This Structure?

- **Separation of concerns**: Each plugin has its own file
- **Easy to find**: Plugin configurations are predictably named
- **Independent**: Plugins can be enabled/disabled by adding/removing files
- **LSP isolation**: LSP configs are separated in their own subdirectory

## Using the Lazy UI

lazy.nvim includes a built-in UI for managing plugins:

```vim
:Lazy
```

This opens an interactive interface showing:
- Installed plugins
- Plugin status (loaded/not loaded)
- Available updates
- Keybindings for actions

<Tip>
Press `?` in the Lazy UI to see all available keybindings.
</Tip>

## Version Locking

The `lazy-lock.json` file at the repository root locks each plugin to a specific commit:

```json
{
  "telescope.nvim": { 
    "branch": "0.1.x", 
    "commit": "a0bbec21143c7bc5f8bb02e0005fa0b982edc026" 
  },
  "nvim-treesitter": { 
    "branch": "master", 
    "commit": "42fc28ba918343ebfd5565147a42a26580579482" 
  }
}
```

### Benefits

- **Reproducibility**: Same plugins, same commits, same behavior
- **Stability**: Updates are controlled and tested
- **Rollback**: Easy to revert to previous versions
- **Collaboration**: Team members have identical setups

<Note>
The lockfile is automatically updated when you sync plugins with `:Lazy sync`.
</Note>
